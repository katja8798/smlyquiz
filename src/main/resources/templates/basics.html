Basics

How to get a SmileyCoin Wallet

The core wallet for Smileycoin is available for several operating systems. For Linux its only available as source code, which is ready to compile and install for Ubuntu. Components may need to be installed for other Linux versions. For Windows and OSX a binary version is available. This is typically a bit behind the Linux version.

An HTML5 wallet is also available at wallet.smileyco.in and runs directly in the browser. Be specially careful to store the passphrase.

For tablets and phones a Coinomi wallet is available (Android and iOS). Searching for Coinomi in the appropriate store should work. This is a stable wallet and appropriate for users not wanting any extra features developed specifically on the SMLY chain (such as on-chain services, voting or traceability).

How to send a transaction

There are several ways to send a transaction, depending on the type of walley you have. You can use the command 'sendtoaddress' in the core wallet command line. If you have a GUI walley, you can simply use the send button. In either case, you need to type in the address you want to send to and the amount.

You can also send a raw transaction. A raw transaction is a transaction in raw bytes. You first need to find an unspent transaction using the command 'listunspent'. Next you create the transaction with createrawtransaction with information about txid, vout, address and amount.

Next you sign the transaction with the command 'signrawtransaction' with the output from createrawtransaction. And finally you can send the transaction with the command 'sendrawtransaction' and the output string from the signiture.

How to set up a multisig address

Multisig allows users to have one smileycoin address with multiple key holders, so funds from the address cannot be spent until all key holders sign off on it. In order to create a multisig address, you will need two or more public public keys to generate it.

We have two methods for generating the address. One is the command createmultisig. Example: createmultisig 3 ‘[“publickey1", "publickey2", "publickey3”]'

The other is the addmultisigaddress command: addmultisigaddress 3 ‘[“publickey1", "publickey2", "publickey3”]'

The output from the command is the “multisig address”. After you´ve sent some amount to the multisig address, you can send transaction from it. All key holders will need to sign the transaction.

How to write data into the blockchain

In principle possible to use all sorts of methods to write data into the blockchain. The most straight forward is to use the createrawtransaction command, which has an optional data parameter. Example: createrawtransaction'[{"txid":"myid","vout":0}]' '{"address":100,"data":" the message you want to write on the blockchain converted in hex"}:'

Then you sign and send the transaction like explained above. After that you can decode the raw transaction using the decoderawtransaction command, and you should see the message you wrote under "asm" : "OP_RETURN “the hex string you wrote”"
